# Phase 3: Decision rigor - Plan

## Goal

cad-context marks only durable decisions (hard-to-reverse, surprising-without-context, the result of a real trade-off) so they resurface via recall while phase-local ones stay put, and a new on-demand `/cad-decision-review <path>` runs a refute-then-adjudicate pass over one decision doc grounded against Context7 and the codebase, producing a per-objection ruling and an amendment list.

## Must be true when done

- A CONTEXT.md written by the updated cad-context places durable decisions under a `## Durable decisions` heading and keeps the phase-local rest under `## Decisions`.
- `planning.mjs recall <terms>` returns a file's `## Durable decisions` items and does not return that file's `## Decisions` items; a legacy CONTEXT.md with only `## Decisions` still resurfaces unchanged.
- `/cad-decision-review <path>` runs only when invoked with a path (never auto-fires) and returns, per objection, a `survives | partial | refuted` ruling plus a concrete amendment list, naming the providers/models/effort that ran.
- During a run the adjudicator verifies at least one library/API claim against Context7 and at least one claim against the codebase.
- With a cross-model provider configured, the run executes the panel through the repaired REV-01 seam, resolving the model from `review.decision_review.{tier,effort}`; single-model (claude-subagent) otherwise.
- `node --test cadence-core/bin/`, `tsc --checkJs`, and `self-verify.mjs` all pass, with the new skill and workflow budgeted, a COMMANDS.md row present, and every new config key prose-referenced.

## Context

Locked decisions from CONTEXT.md bind this plan: durability is prose judgment in cad-context, no scoring seam (D-01); durable decisions get a new `## Durable decisions` heading with `## Decisions` holding the phase-local rest (D-02); a legacy file with no durable heading has its `## Decisions` read as durable (D-03, no retrofitting legacy phase 1/2 files). DEC-02 ships as a new skill + workflow reusing the review subsystem (D-04/D-05/D-06); the `survives|partial|refuted` ruling is adjudicator prose output, not a FINDING_SCHEMA or CONTRACTS change (D-07); Context7 + codebase grounding sits on the skill/main-model surface, not the read-only cad-reviewer (D-08); cost reporting is qualitative (D-09); config is `review.decision_review.{tier,effort}` (D-10); on-demand only, no trigger wiring (D-11); the durability→review link is human judgment, no mechanical handoff (D-12). Follow existing patterns: cad-plan-review/SKILL.md + review-triggers.md for the review path, cad-context/SKILL.md + workflows/context.md for the skill→workflow @execution_context pattern, review.consult.* for the config group shape. Out of scope: no ADR tree, no glossary, no auto-selection, no review-provider.mjs schema change.

## Tasks

### Task 1: Retarget the recall parser to `## Durable decisions` with a legacy `## Decisions` fallback

- **Files:** cadence-core/bin/lib/planning-files.mjs, cadence-core/bin/planning.test.mjs
- **Action:** In `parseContextDecisions`, change the section source: read `sectionBody(text, 'Durable decisions')` first; fall back to `sectionBody(text, 'Decisions')` **only when the durable read returns `null`** (the heading is absent). Do NOT fall back on an empty string: `sectionBody` returns `null` only when the heading is missing, but returns `""` when the heading is present with no items, so the naive `!durable` / `durable || sectionBody(...)` idiom (the one the existing `if (!body) return []` line uses) is a trap here - use a strict `durable === null` test (or `durable ?? sectionBody(...)`, since `??` coalesces only nullish, not `""`). Keep the existing `- D-NN` line filter and `- ` strip unchanged, so a global D-NN sequence spanning both sections still indexes only lines under whichever section was selected. The presence of a `## Durable decisions` heading means the file uses the v1.2 convention and its `## Decisions` items are phase-local and must not be indexed, even when the durable section holds zero D-NN lines. Update the function's doc comment to state the durable-first, null-only-fallback rule. In planning.test.mjs, extend the `makeTree` fixture's phase spec: add a `durableDecisions` array that writes a `## Durable decisions` section of `- D-NN (area): text` lines, alongside the existing `contextDecisions` which continues writing `## Decisions`; when both are given the file carries both sections with a continuous D-NN numbering. Distinguish the two zero-durable shapes: a `durableDecisions: []` empty array writes a present-but-empty `## Durable decisions` heading (the v1.2 all-phase-local shape), while omitting `durableDecisions` entirely writes no durable heading at all (the legacy shape). Add four tests: (1) durable-parse - a phase with `durableDecisions:['use foobar approach']` and `contextDecisions:['phase-local baz detail']` where `recall('foobar')` returns a result sourced `phases/N/CONTEXT.md` and `recall('baz')` returns no CONTEXT result; (2) legacy-fallback - a phase with only `contextDecisions` (no durable heading) still resurfaces via recall unchanged; (3) empty-durable - a phase with `durableDecisions: []` (present-but-empty durable heading) plus a populated `contextDecisions:['phase-local baz detail']` where `recall('baz')` returns no CONTEXT result, proving the parser does NOT fall back to `## Decisions` on a present-but-empty durable section (this is the common all-phase-local shape and the case a `!durable` mis-implementation silently breaks while passing tests 1-2); (4) determinism - two `recall` runs over a corpus containing a `## Durable decisions` section are byte-identical.
- **Verify:** `node --test cadence-core/bin/planning.test.mjs` passes including the four new tests - and specifically the empty-durable test fails against a `!durable`/`||` fallback and passes only against the `=== null` / `??` rule; `npx tsc -p tsconfig.ci.json` (the checkJs config) reports no new errors.
